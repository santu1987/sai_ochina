About Visual QuickMenu
Visual QuickMenu packs full featured menu design capabilites into a compact and simple interface. The interface is pure client side JavaScript which is designed to run identically in both IE and FireFox. The menus produced are full cross browser and identical to the text_based version of QuickMenu.

Visual QuickMenu Basics
  • The QuickMenu interface runs on-top of any existing QuickMenu inside the browser, this means that if you publish a template menu to your webpage you can design while viewing your page in iether FireFox or IE. This works regardless of wheather your page is local or live.
  • All file references are relative to the HTML document you are working in. The no-hassle way to ensure correct imgage references is to use absolute paths (e.g. http://www.mysite.com/images/myimage.gif). If your design does not allow for absolute paths, customize the menu from within a mirrored folder structure of your web site, this way all images will be visible during design.
  • Be carefull not to refresh or close the browser window during design, or you will lose your work! Because the program is browser based there is no way to stop the browser from moving on, so save often if this is of concern.
  • Quick Menu and the Visual Interface use pure client side cross browser JavaScript. You may use any method of server side content delivery you wish, including database generated items, includes, etc.
  • You may manually customize the menus produced by the visual tool with a plain text editor, your manual changes will be recognized by the visual interface if you switch between text based and visual edits.
